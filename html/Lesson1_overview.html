Lesson 1: Overview

bob-angular is an ambitious tool for learning a number of current technologies and techniques through functional examples. Rather than focus on a single application or framework, bob-angular looks at the interconnectedness of the many moving parts in a modern web application. To do this, it includes an express server and postgres database, as well as a fully functional angular application.

Initial Concept

In creating this seed, I wanted to acquire new knowledge, apply existing knowledge to something that could capture my imagination, and give enough dimension and thought to it to make it useful to others. As a matter of purpose, design choices were made to include:

  • a rich single page app run on top of angular
  • a modern server side component built on top of node
  • a development and build process that included modern automation and testing
  • a demonstration of traditional 2d web content mapped to animating 3d surfaces
  • methods to provide javascript as well as css support to responsive design
  • a youtube feed component
  • an open source mapping component
  • an svg interactive map of the world
  • a functioning server and sample database integration
  • a method to push builds to github and then heroku

Most of the larger objectives were included at the outset, but as development began, new ones were identified and included as possible. At the end of this initial rev, there are known (and unknown) anomalous functionalities that will want to be improved, but there is at least enough to sink one's teeth in without worry of acquiring too bad a taste from its implementation.

Technology Choices

The following client and server technologies, libraries and frameworks have been used in the creation of bob-angular.

Client technologies

  • html5, css3 and javascript
  • angular.js
  • three.js
  • less
  • bootstrap
  • jquery
  • d3
  • leaflet
  • youtube
  • html5 boilerplate

Server technologies

  • node
  • express
  • postgres
  • grunt
  • bower
  • jasmine
  • protractor
  • karma
  • git
  • heroku

As much as possible, attempts were made to create structures and make code and process choices that would enable late-conceived features to be included without impacting the overall application. The introduction will conclude with a brief description of the application's file and directory structure.

File and Directory Structure

In the outermost directory, there are five sub directories and nine files for installation and server tasks. The client application is built during the installation process and is a child of the "dist" directory. 

	app/                                   --> application source files
	  assets/                              --> media, CSS JS, JSON assets
	  components/                          
	    footer                             --> view component
	    main                               --> view component
	    context                            --> view component
	    navigation                         --> view component
	    youtube                            --> feature component
	    leaflet                            --> feature component
	    svg                                --> feature component
	  shared                               
	    three_module                       --> feature service
	    country_module                     --> feature service
	  libs/                                --> repository for bower-created 
	    3l, angular, bootstrap, three.js,      source files from 'npm start' or  
	    jquery, less, ui-router, others        'bower install' 
	  app.js                               --> main application module
	  index.html                           --> the entrace point to the app.

	dist/                                  --> grunt-created after bower install

	*****   IMPORTANT!     *****           --> dist contains compiled app/ files

	node_modules/                          --> created for server use during install
	tests/                                 --> supporting files for tests 
	tests/e2e-tests/                       --> end-to-end tests
	.bowerrc                               --> directs bower to load libs
	.gitignore                             --> the files that are excluded for git
	bower.json                             --> client-side libraries to install
	exports.index.js                       --> express server for testing
	Gruntfile.js                           --> Creates the files that go in dist
	index.js                               --> express server and postgres gateway
	karma.conf.js                          --> config file for karma unit tests 
	package.json                           --> instruction files for npm 
	Procfile                               --> runs _node index.js_ on heroku
